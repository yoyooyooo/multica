# Features

## Owns

- Current capability narratives and user/operator-visible semantics implemented in this generation.
- Capability boundaries, failure behavior, non-goals, and source/test anchors.

## Must not own

- Branch, build, deployment, or review procedures.
- Active tracker/PR status.
- Future capabilities not implemented in the current generation.
- Runtime availability claims without deployment evidence.
