# Releases

## Owns

- Immutable release and fork-generation manifests.
- Accepted source identity, verification scope, artifact references, rollout evidence links, rollback source, and explicit claim limits.

## Must not own

- Active implementation or deployment progress.
- Product or architecture authority.
- Raw logs, credentials, or mutable environment snapshots.

## Read next

- [Fork generation manifests](fork-generations/README.md)

Tracker Issues and PRs own active work. A release manifest records accepted facts after their evidence exists.
